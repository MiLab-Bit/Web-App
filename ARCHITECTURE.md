```

        │
        ▼
┌───────────────────┐

└────────┬──────────┘
         │
         ▼
┌───────────────────┐

│ (getServerSession)│

    ▼         ▼

    │         │

  重定向到
  /dashboard
```

## 三、安全性设计

### 3.1 密码安全

```typescript
// 注册时：密码哈希
import bcrypt from 'bcryptjs';

const hashedPassword = await bcrypt.hash(password, 10);
// 存储 hashedPassword 到数据库

// 登录时：密码验证
const isValid = await bcrypt.compare(inputPassword, storedHash);
```

### 3.2 Session 安全

```typescript
// JWT 配置
{
  session: {
    strategy: 'jwt',
    maxAge: 30 * 24 * 60 * 60, // 30 天
  },
  cookies: {
    sessionToken: {
      name: 'next-auth.session-token',
      options: {
        httpOnly: true,      // 防止 XSS 攻击
        sameSite: 'lax',     // CSRF 保护
        path: '/',
        secure: true,        // 仅 HTTPS (生产环境)
      }
    }
  }
}
```

### 3.3 CSRF 保护

NextAuth.js 自动处理 CSRF Token：
- 每个表单提交都包含 CSRF Token
- API 端点验证 Token 有效性
- 无需手动配置

这个架构设计确保了：
1. **安全性**: 密码哈希、JWT、CSRF 保护
2. **可扩展性**: 模块化设计，易于添加新功能
3. **性能**: SSR、代码分割、缓存策略
4. **可维护性**: TypeScript 类型安全、清晰的目录结构
5. **开发效率**: Next.js 全栈一体化，减少上下文切换
